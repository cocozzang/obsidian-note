## 📁 소스 자료 정보

### 원본 강의 자료

**형식**: PDF → PNG 이미지 변환 완료

**경로**: 정리파일 작성 요청시 대화창에 이미지 전달

### 이미지 활용 방법

#### 1️⃣ 파일 작성 시 이미지 참조 프로세스

```bash
# 1단계: 해당 강의의 페이지 범위 확인
# 예: 1강 (p.1-3) → 페이지-1, 페이지-2, 페이지-3

# 2단계: 프로젝트 공유파일 영역에서 강의자료 확인하기
# 프로젝트 공유영역내의 강의 자료들의 순서를 확인해야함
# 이미지 하단에 페이지 수가 숫자로 제공됨

# 3단계: Claude가 모든 이미지를 순차적으로 분석
# - 수식 인식 및 LaTeX 변환
# - 정의, 정리, 예제 구분
# - 그래프/도형 → ASCII art로 변환

# 4단계: 분석한 내용을 템플릿에 맞춰 마크다운 작성
```

#### 2️⃣ 이미지 분석 체크리스트

**각 이미지에서 추출할 정보**:

- [ ] 강의 제목 및 소제목
- [ ] 수학적 정의 (Definition)
- [ ] 정리 (Theorem) 및 증명
- [ ] 공식 (Formula)
- [ ] 예제 문제 및 풀이
- [ ] 그래프, 도형, 표
- [ ] 주의사항 및 참고
- [ ] 연습문제

#### 3️⃣ 수식 변환 가이드

**이미지 내 수식 → LaTeX 변환 원칙**:

- 인라인 수식: `$...$`
- 블록 수식: `$$...$$`
- 분수: `\frac{a}{b}` (dfrac 대신 frac 사용)
- 극한: `\lim_{x \to a}`
- 적분: `\int_{a}^{b}`
- 그리스 문자: `\alpha, \beta, \theta` 등
- 복잡한 수식: 여러 줄로 분할하여 가독성 확보

#### 4️⃣ 그래프 처리 방법

**기본 원칙: ASCII 아트로 표현** ✅

모든 그래프는 ASCII 아트로 표현합니다.

**표준 형식**:

```markdown
<!-- 그래프: [설명] -->
```

[ASCII 아트 그래프]

````

**예시 1: 극한의 정의 그래프**

```markdown
<!-- 그래프: 극한의 기하학적 의미 -->
````

    y
    |

L |........● y = f(x)
| /
| /
| ●
| /
----+----●---------●---- x
-1 0 1 a 2

````

**예시 2: 좌극한과 우극한**

```markdown
<!-- 그래프: 좌극한 L⁻과 우극한 L⁺ -->
````

    y
    |          y = f(x)

L⁺ | ○-------●------
| / |
L⁻ |--● |
| |
----+---------●--+------- x
a

```

**ASCII 아트 작성 가이드**:

1. **기본 좌표축**

```

y
|
|
+---------- x

````

2. **점 표시**

- 채워진 점: `●`
- 빈 점 (열린 구간): `○`
- 좌표 라벨: 축 옆에 표시

3. **곡선 표시**

- 직선: `/`, `\`, `|`, `-`
- 곡선: 연속된 점이나 문자로 근사
- 점선: `.` 또는 `:`

4. **주요 값 표시**
- 극한값 L, 함수값 f(a) 등
- 특이점 명시

#### 5️⃣ 작성 워크플로우

**Step 1**: 프로젝트 공유폴더 영역의 강의 이미지 확인

- 이미지 하단의 페이지 번호로 순서 파악
- 해당 강의 범위의 모든 이미지 로드

**Step 2**: 모든 이미지 순차적으로 분석

- 정의, 정리, 예제 구분
- 수식을 LaTeX로 변환
- 그래프를 ASCII 아트로 변환

**Step 3**: 내용 구조화

- 강의 주제별로 그룹핑
- 예제와 풀이 분리

**Step 4**: 템플릿에 맞춰 마크다운 작성

- YAML 메타데이터 작성
- 본문 내용 작성
- 예제 풀이는 토글 형식으로 작성

### ⚠️ 주의사항

1. **이미지 분석 정확도**: 수식은 반드시 원본 이미지와 대조하여 검증
2. **이미지 순서**: 페이지 하단의 번호로 순서 확인
3. **수식 일관성**: LaTeX 표기법 통일 (`\frac` 사용)
4. **그래프는 ASCII로**: 모든 그래프는 ASCII 아트로 표현
5. **예제 풀이 형식**: 반드시 Obsidian 콜아웃 토글 사용

## 📖 파일 작성 가이드

### 파일명 규칙

**형식**: `[강의번호]-[영문-주제]-[순번].md`

- 강의번호: 2자리 숫자 (01, 02, ..., 65)
- 영문-주제: 소문자, 하이픈으로 연결
- 순번: 같은 주제의 연속 강의인 경우 -1, -2, ... 추가

**예시**:
- `01-function-limit-1.md` (1강: 함수의 극한(1))
- `02-function-limit-2.md` (2강: 함수의 극한(2))
- `08-advanced-trigonometry-1.md` (8강: 심화삼각함수(1))

### YAML 메타데이터 작성 규칙

```yaml
---
id: [UUID v4 형식]

- [강의 한글명]

---
````

**필수 필드**:

- `id`: UUID v4 형식 (예: `4347c979-da80-4b1b-9cba-9350f4094f0f`)

- `pages`: 페이지 범위 (예: `1-3`)
- `tags`: 한글 태그 사용 (영문 태그 ❌)
- `topics`: 해당 강의에서 다루는 주제 목록

### 예제 풀이 작성 규칙

**필수**: Obsidian 콜아웃 토글 형식 사용 ✅

```markdown
### 예제 1

[문제 내용]

> [!summary]- 풀이
> [풀이 단계 1]
>
> [수식이나 계산]
>
> [풀이 단계 2]
>
> [결론]
```

**특징**:

- `> [!summary]- 풀이`: Obsidian 콜아웃 토글 형식
- `-` (마이너스): 기본적으로 접혀있음
- 클릭하면 펼쳐지고, 다시 클릭하면 접힘
- 학습자가 스스로 문제를 푼 후 정답 확인 가능

**콜아웃 타입**:

- `[!summary]`: 요약/풀이 (주로 사용)
- `[!note]`: 참고사항
- `[!tip]`: 팁/힌트
- `[!warning]`: 주의사항

## 📐 콘텐츠 구성 방침

### 수식 표현

- LaTeX 문법 사용
- 인라인: `$...$`
- 블록: `$$...$$`
- 분수: `\frac{a}{b}` (dfrac ❌)
- 극한: `\lim_{x \to a}`
- 중요 공식은 별도 블록으로 강조

### 그래프/다이어그램

- ASCII 아트 활용
- 좌표평면, 함수 그래프 시각화
- 점, 선, 곡선을 문자로 표현

### 예제

- **필수**: `> [!summary]- 풀이` 토글 형식
- 단계별 풀이 과정 제시
- 핵심 개념 적용 명시

### 연습문제

- 각 챕터당 학습 포인트 제시
- 복습 가이드 제공

### 내부 링크

- Obsidian 내부 링크: `[[파일명|표시명]]`
- 예: `[[02-function-limit-2|함수의 극한 (2)]]`
- 관련 개념 상호 참조
- 이전/다음 강의 연결
